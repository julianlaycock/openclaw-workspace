Executive SWOT

Strategic design positioning relative to market

✦ Strengths

Content depth is exceptional — 60+ API endpoints, Annex IV knowledge, XSD validation, sanctions screening. No competitor has this much substance on their landing page. Typography system (Sora/Inter/JetBrains Mono) is a strong triplet. Color palette is unique in the compliance space (teal + warm coral vs. generic blue). Performance is outstanding (152ms load, 77KB single-file HTML). Blog content exists and ranks.

✦ Weaknesses

No logo/mark — just a text wordmark. No favicon (generic). No brand guidelines document. Landing page hero lacks a visual anchor (just text + terminal animation). Reveal animations are broken on fresh load (opacity stuck at 0). Login page is bare minimum. Demo redirects to login on production. No dark/light toggle. Dashboard sidebar has no visual hierarchy refinement. Blog is behind auth on production. No 404 page design.

✦ Opportunities

Direct competitor (anevis) has 2005-era design — stock photos, blue corporate template, zero interaction design. This is the lowest bar in enterprise compliance. Sphinx ($7.1M) has a clean site but it's generic. With a proper 3D hero render + brand mark + micro-interactions, Caelith instantly becomes the most visually premium compliance product in EU fund management. The v2a sphere render prototype shows this is achievable.

✦ Threats

Mojibake and broken animations damage credibility instantly. A compliance officer evaluating Caelith will see corrupted characters and blank sections — and close the tab. The "Gewerbeanmeldung in Vorbereitung" on the impressum signals pre-revenue. No social proof (testimonials, logos, case studies). The countdown timer ("46 days") will become stale and needs dynamic updating.

Strategic Design Roadmap

Prioritized by impact × effort — highest ROI first

Phase 0
This weekend
  • Fix broken reveals — run the fix script, verify all 32 elements animate on scroll. 30 minutes, highest impact single fix.
  • Fix mojibake — run fix-mojibake.mjs. 5 minutes. Eliminates credibility damage.
  • Fix demo funnel — /demo-dashboard must work on production or the CTA must redirect to something that does (e.g. Calendly).
  • Fix blog index — /blog is behind auth on production. Blog content is a growth engine that's currently locked.
  • Fix lang toggle — DE version links to ?lang=de for both options (should link to /api/landing for EN).
Phase 1
Week 1
  • Brand mark / logo — Commission or design a shield-based mark. This appears on favicon, OG images, login, docs, pitch deck. Highest brand ROI.
  • Hero 3D render — Integrate the noise sphere or glass shield render into the hero section. This single change moves hero impact from 5/10 to 8/10.
  • OG image card — Branded gradient card with logo + tagline for social sharing. Every LinkedIn/Twitter share becomes brand advertising.
  • Login page redesign — Add gradient background, glass card, brand mark. 2 hours of work, massive perceived quality improvement.
Phase 2
Week 2-3
  • Social proof section — Even without customers: "Backed by [accelerator]", "Selected for Campus Founders Batch 4", EXIST grant logos, "Built on ESMA standards" trust bar. Something > nothing.
  • Landing page section consolidation — Merge 14 sections to 9-10. Remove or combine: open source bar + blog can merge, comparison table can fold into features, process section is redundant with the API section.
  • Micro-interactions audit — Add hover states to every card, loading transitions, page transitions, scroll progress indicator. This is what separates "functional" from "premium."
  • Custom Swagger theme — Brand the API docs with Caelith colors, fonts, custom header. 4 hours, instantly more professional.
Phase 3
Week 4-6
  • Design system unification — Bridge the landing page CSS and dashboard Tailwind into a single token system. Document in a brand guidelines page.
  • Dashboard UI polish — Sidebar active states, data visualization (timeline charts, risk heat maps), skeleton loading, empty state illustrations.
  • Mobile experience — Animated mobile nav, swipeable pricing cards, tablet breakpoint, touch-optimized interactions.
  • Accessibility audit — Full WCAG 2.1 AA compliance. Contrast fixes, screen reader testing, keyboard navigation, reduced-motion support.
  • 404 / error pages — Branded error pages with helpful CTAs. Currently shows default Next.js 404.
Phase 4
Week 7+
  • Interactive demo — Sandboxed product tour that works without auth. This is the #1 conversion lever for B2B SaaS. Sphinx has this. Linear has this.
  • Blog redesign — Proper blog index with cards, categories, featured posts. Currently just raw article pages.
  • Testimonial / case study system — As customers come in, have the infrastructure ready for quotes, logos, and case studies.
  • Dark/light toggle — The landing page is dark-only, dashboard has both themes. Give users control everywhere.

Final Verdict

Caelith at 60/100 is not a design failure — it's a design debt problem. The product has genuine substance that competitors can't match. The typography, color system, and content quality are above average. But the brand identity gap (no logo, no mark, generic OG images) and the broken user paths (demo dead-end, blog behind auth, reveal animations stuck) undermine the credibility that the content works so hard to build.

The biggest opportunity: Your direct competitor (anevis) has 2005-era design. Sphinx has clean-but-generic design. Neither has the technical content depth you have. If you execute Phase 0-2 (brand mark + hero render + social proof + fix bugs), you jump from 60 to ~78 in two weeks — and become the most visually premium compliance platform in European fund management.

The non-negotiable fix: Run the bug fix scripts today. Broken reveal animations and mojibake characters are instantly disqualifying for a compliance product that sells trust. Everything else can wait; this can't.
