Color & Chart Harmonization Audit

Every visualization in Caelith, current inconsistencies, and a unified palette proposal.

1. Current Problems Found

🔴 Two completely different color systems

Fund detail (InvestorTypeDonut): Uses brand palette — #C5E0EE, #E8A87C, #B8C9A3
Dashboard (InvestorTypeBreakdownCard): Uses Tailwind defaults — #3B82F6 blue, #10B981 green, #F59E0B amber, #EF4444 red
Same data, completely different look.

🔴 Jurisdiction colors conflict

Fund detail (JurisdictionExposureBar): DE = #C5E0EE, LU = #E8A87C
Dashboard (JurisdictionExposureCard): DE = #3B82F6, AT = #10B981, CH = #F59E0B, LU = #8B5CF6
Germany is two different colors depending on which page you're on.

🟡 Three different "green" meanings

#6ee7b7 (emerald-300) in dashboard score ring
#10B981 (emerald-500) in investor type breakdown
#16a34a (green-600) in CSS --success variable
#bbf7d0 (green-200) in fund bands
Same semantic "good/compliant" = 4 different greens.

🟡 Semantic color overlap

#E8A87C means "warm accent" in brand palette but also "warning/medium severity" in charts. #F59E0B (amber) is used for semi_professional investors AND as a warning indicator. Colors should mean one thing.

🟡 Chart component duplication

Two investor type charts: InvestorTypeDonut (treemap) on fund detail + InvestorTypeBreakdownCard (bars) on dashboard. Two jurisdiction charts: JurisdictionExposureBar + JurisdictionExposureCard. Should be one component, one style.

🟡 Hardcoded vs CSS variable mix

Dashboard uses raw hex (#fca5a5, #6ee7b7). Charts use var(--success), var(--danger). Some use WARM const. No single source of truth.

2. Proposed Unified Color Palette

Semantic Colors (status/severity)

One green, one amber, one red. Used everywhere for the same meaning.

SUCCESS#6ee7b7
WARNING#E8A87C
DANGER#f87171

✓ #6ee7b7 = compliant, verified, approved, pass, low risk — THE green
✓ #E8A87C = warning, medium, expiring, caution — THE amber/warm
✓ #f87171 = danger, critical, rejected, expired, high risk — THE red

Data Categorical Colors (investor types, jurisdictions, etc.)

Derived from brand palette. Harmonious, distinguishable, not confused with semantic colors.

CAT-1#C5E0EE
CAT-2#A3C4D4
CAT-3#7BA8BD
CAT-4#5A8FA6
CAT-5#3D7A91
CAT-6#2A6478

All derived from brand accent #C5E0EE — monochromatic blue scale.
No orange, no random Tailwind blue, no purple. One family of colors for all categorical data.
