## 1. Critical Bug: localStorage Key Mismatch
**Impact:** Date, number, and currency formatting ignored the user's locale setting.

| File | Before | After |
|------|--------|-------|
| `utils.ts` (5 occurrences) | `localStorage.getItem('caelith-locale')` | `localStorage.getItem('caelith_locale')` |

`i18n.tsx` uses `caelith_locale` (underscore), but `utils.ts` used `caelith-locale` (hyphen). This meant `formatDate()`, `formatNumber()`, `formatCurrency()` always defaulted to 'de' regardless of user setting.

## 2. formatPercentage Locale Support
**Before:** `formatPercentage()` hardcoded English decimal point (`n.toFixed(2) + '%'`)  
**After:** Uses `Intl.NumberFormat` with locale detection → German gets `72,50%` not `72.50%`
