01 — Elevator Pitch

The 30-Second Version

Caelith is the compliance infrastructure layer for European alternative investment funds. We turn the manual, error-prone process of AIFMD regulatory reporting — which costs fund managers €150-200K per year — into automated, XSD-validated API calls. With AIFMD II enforcement hitting April 16, 2026, and 47% of fund managers unprepared, we're building the open-core compliance stack that both human teams and AI agents rely on to file, screen, validate, and audit.

"We're doing for EU fund compliance what Stripe did for payments — infrastructure that makes the hard stuff a single API call."

02 — The Problem

Why This Exists

What is AIFMD II? The EU's updated rulebook for anyone managing alternative investment funds — hedge funds, real estate, PE, infrastructure. Rewrites reporting, adds delegation disclosures, mandates liquidity management tools, expands Annex IV data points.

The deadline: April 16, 2026. Member states must transpose into national law. Germany's version (FRiG) amends KAGB. Miss compliance → fines, license risk, investor lawsuits.

How it's done today:

• Compliance officers manually fill XML/Excel templates (15-30 hours per filing)
• Consultants charge €3K-5K per filing, or €150-200K/year ongoing
• Fund admins bundle it into fees with zero transparency
• Errors get rejected by BaFin's MVP-Portal — no pre-validation exists

"Right now, a compliance officer at a small German KVG spends two days filling in an XML template by hand. One typo and BaFin rejects the filing. We eliminate that entire process."

03 — The Product

12 Features, Explained

For each: what it does, why it matters, how it works, and what to say in the meeting.

01 Rules Engine

Plain English13 compliance rules across 6 frameworks (KAGB, AIFMD, MiFID II, SFDR, DORA, AML), evaluated in real-time against fund and investor data. Catches violations before they happen.
Under the HoodEach rule is a typed function evaluating fund metadata + investor profiles. Deterministic results: pass/fail + decision explanation + regulatory citation. Fully auditable.

"Our rules engine evaluates 13 compliance rules in real-time — an investor's eligibility is checked against KAGB, AIFMD, MiFID II, and AML simultaneously, with full audit trail."

02 Cryptographic Audit Trail

Plain EnglishEvery compliance decision is SHA-256 hashed and chained to the previous entry — a tamper-evident log. DORA and KAGB require documented, audit-proof decisions.
Under the HoodEach decision_record hashes payload + previous hash (blockchain-like chain). Any modification breaks the chain. Immutability enforced at PostgreSQL trigger level.

"Every decision is cryptographically hash-chained. If a regulator asks 'show me the decision trail,' we can prove no record was tampered with — mathematically."

03 Annex IV XML Reporting

Plain EnglishGenerates fully structured AIFMD Annex IV XML, validated against ESMA's official Rev 6 XSD schema. This is the actual format BaFin's portal accepts.
Under the HoodSerializer maps fund data → 200+ XML fields, handles all report types (24_1, 24_2, 24_2_4, 24_4). Validates against downloaded ESMA XSD. 29 schema errors found and fixed.

"We validate Annex IV XML against ESMA's actual XSD schema before you file. We found and fixed 29 schema errors that would have been rejected by BaFin."

04 Sanctions Screening

Plain EnglishScreens entities against 6,863 EU + UN sanctioned persons/orgs using fuzzy name matching. AML screening is mandatory before investor onboarding.
Under the HoodParses EU + UN sanctions XML into PostgreSQL. Uses pg_trgm (trigram matching) for fuzzy comparison — catches transliteration variants, typos, aliases. Configurable threshold.

"We screen 6,863 EU and UN sanctioned entities with fuzzy matching — 'Vladimir' still matches 'Wladimir.' Milliseconds, not days."

05 LEI Verification

Plain EnglishValidates Legal Entity Identifiers in real-time via GLEIF API. Invalid LEIs = rejected Annex IV filings. LEIs expire, entities merge, statuses change.
Under the HoodCalls GLEIF's free API, caches results, validates 20-char alphanumeric + checksum format, checks active/inactive/merged status. Auto-populates entity name.

"Every LEI is validated against GLEIF in real-time. No more rejected filings because an LEI expired three months ago."

06 EMT / EET / EPT Templates

Plain EnglishAuto-generates European MiFID Template, ESG Template, and PRIIPs Template — 43+ fields pre-filled. Distributors require these to sell funds.
Under the HoodMaps fund metadata (NAV, fees, risk, ESG classification) to FinDatEx standardized fields. Exports CSV/XLSX.

"EMT, EET, EPT — the three templates every distributor demands. We auto-generate all three from your fund data."

07 Compliance Copilot AI

Plain EnglishAI assistant answering regulatory questions in context — 80+ terms in EN/DE — with live access to your actual fund data. Not generic ChatGPT answers.
Under the HoodTool-use architecture on Claude Haiku. 16-table SQL allowlist prevents leakage. Prompt injection defense. Queries live fund/investor/LMT data. PII-sanitized. ~€0.02/question.

"Ask 'Are we compliant with Art. 16 LMT requirements?' — it checks your actual fund data and tells you exactly what's missing."

08 XSD Validation

Plain EnglishPre-submission validation against ESMA's official schema. Catches structural errors before filing — no more BaFin rejections.
Under the HoodRuns full schema validation on generated XML, reports line-level errors with human-readable descriptions. Took 29 iterations to perfect.

"We validate against the exact same schema BaFin uses. If it passes our check, it passes theirs."

09 Developer API

Plain EnglishFull REST API with OpenAPI/Swagger docs, versioned routes, API key auth. Fund admins embed compliance checks into their own workflows.
Under the Hoodck_live_ prefix keys, bcrypt hashed. Tiered rate limiting. Backward-compatible versioning. Swagger at /api/docs. Every dashboard feature = API endpoint first.

10 AIFMD II Readiness Check

09 — Objections

Common Pushback & Responses

"Isn't this just a dashboard?"
The dashboard is a reference implementation. The core product is API infrastructure — filing pipelines, validation engines, screening services. Fund admins don't want dashboards; they want APIs they embed. That's where the scale lives.
"How is this different from anevis?"
anevis is managed service for enterprises — email them, they do the work, €15-50K. We're self-service infrastructure at 10x lower cost. Different customers, different model. They're Accenture, we're AWS.
"Do KVGs even buy software?"
Real competitor isn't other tools — it's Excel and inertia. But AIFMD II changes the math: 3x more data points, stricter validation, new sections. Excel doesn't scale. April 2026 forces action.
"Solo founder risk?"
Valid concern. That's why I'm pursuing EXIST for structured support, HWR for academic backing, and building open-core — the community becomes the team. Plus Lanzadera is behind us.
"Too early for us" (VC)
Completely understand. What specific milestones — revenue, customers, product — would make us interesting for a future conversation?
"What about risk calculations?"
anevis integrates VaR/DV01 — we don't yet. But AIFMD II's new requirements are delegation disclosures, LMT documentation, and loan origination reporting. Those are our focus. Risk calc is Phase 2.
"Will ESMA delay the deadline?"
~30% chance. But even current Annex IV is manual and painful. We solve today's problem AND tomorrow's. No wasted investment either way.

11 — Infrastructure Pivot

APIs for Fund Admins — Explained

The Shift: Dashboard SaaS → Compliance Infrastructure

Every AI company can build a dashboard. Infrastructure retains value. Think Stripe: merchants never see Stripe's dashboard — they use the API. The dashboard is a demo. The API is the business.

Caelith's pivot: stop selling a compliance dashboard and start selling compliance infrastructure that fund admins embed into their own systems.

What Goes Through the API

Four real endpoints, each replacing weeks of manual work:

API Annex IV Generation

POST /api/v1/annex-iv/generate

Send fund data → get XSD-validated Annex IV XML back. What took 2 weeks of manual work now takes 2 seconds.

API Sanctions Screening

POST /api/v1/sanctions/screen

Send a name → get fuzzy match results against 6,863 EU + UN sanctioned entities. Milliseconds, not days.

API LEI Validation

GET /api/v1/lei/validate/{LEI}

Validate any LEI's status against GLEIF in real-time. No more rejected filings from expired identifiers.

API EMT Template Generation

POST /api/v1/templates/emt

Send fund metadata → get a fully filled European MiFID Template. Covers EMT, EET, and EPT.

Why Fund Admins Care

Fund administrators manage 50-200 fund clients each. They do the same compliance work 200 times. One API integration serves all their clients instantly.

What took a compliance team 2 weeks per fund now takes 2 minutes. One fund admin deal = €50-200K ARR.
