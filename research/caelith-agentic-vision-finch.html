The Problem
Every EU fund manager faces the same broken process.

AIFMD requires fund managers to file detailed regulatory reports every quarter, semi-annually, or annually — per fund, per NCA, in jurisdiction-specific XML formats. Today this is done manually.

📋

300+ data fields, mapped manually

40–80h consultant per filing

BaFin, CSSF, AMF, FMA and 4 others each add their own layer on top of ESMA's framework — different portals, different field interpretations, different deadlines. One mistake means rejection.

Specialist knowledge per NCA

The Insight
Compliance isn't a judgment problem. It's an execution problem.

Every compliance workflow in EU fund management follows a deterministic policy: ESMA defines the schema, KAGB defines the classification rules, BaFin defines the submission format. There is no ambiguity in the process — only in executing it across hundreds of fields, across multiple jurisdictions, without making mistakes. This is exactly the kind of work autonomous agents are built for: rule-based, high-stakes, and currently being done by expensive humans who should be spending their time on the 5% of cases that genuinely require judgment.

Caelith's architecture reflects this: agents handle the deterministic execution. Humans approve at two explicit gates. The audit trail writes itself.

The Architecture
One persistent conversation thread.
Five agents working in the background.

The compliance officer never navigates a UI. They open one chat, get a briefing on what needs attention, and approve at key decision points. The rest is autonomous.

1

Intelligence Layer — Always watching, always on

Runs silently in the background. Monitors regulatory calendar deadlines (42d → 14d → 7d), sanctions list updates (rescreens all investors on every EU/UN list change), ESMA and BaFin circulars, LEI status changes, and portfolio threshold triggers. Only surfaces events that actually require attention.

Continuous Zero-action from CO
2

Surfacing Layer — Context-loaded briefings, not notifications

When an event matters, the agent surfaces it in chat pre-loaded with all relevant context: fund data, pre-flight score, investor health, last filing status. The compliance officer's first message has everything needed to decide — no back-and-forth to gather context.

Priority-ranked Memory-aware
3

Conversation Layer — Natural language in, regulatory filings out

The CO types naturally. The agent parses intent, routes to the right execution engine, and asks only for what it cannot find itself — one question at a time. Two explicit approval gates: one to generate the XML filing, one to submit to the NCA portal.

Intent classification 2 approval gates
4

Execution Layer — Deterministic, not probabilistic

Every action is a typed call against a known schema. GLEIF returns a verified LEI fact. XSD validation returns pass/fail against ESMA AIFMD_DATAIF_V1.2.xsd. BaFin portal returns a submission reference number. No hallucination risk — execution is deterministic by design. SHA-256 chained audit trail written automatically at every step.

Why Now
Three forces converging in 2026.

⏳ AIFMD II deadline: April 16, 2026

AIFMD II transposition is mandatory across all 27 EU member states by April 16. New requirements around sub-delegations, depositary oversight, fees disclosure, and liquidity management tools make the manual process significantly more complex and error-prone. Fund managers who aren't already updating their Annex IV workflows are running out of time.

🤖 Agent infrastructure is finally production-ready

Browser automation (Playwright), structured output APIs, and reliable function-calling have reached the maturity needed for deterministic compliance workflows. 12 months ago this would have meant hallucination risk in regulatory outputs. Today it means fast, accurate, auditable execution — with the LLM handling conversation and routing, and typed APIs handling the actual data.

🌍 NCA digitisation is accelerating

Norway mandated XML-only Annex IV submissions from June 2026. BaFin's MVP Portal continues modernising. CSSF eDesk launched a new programmatic interface. The portals are becoming automatable — and the compliance teams expected to use them are still manually filling Excel templates. The gap is widening.

📈 Scale demand is rising

Fund administrators managing 15–50+ mandates need to file Annex IV for every fund, every cycle. That's 60–200 filings per year from a single firm. Linear headcount scaling doesn't work at this volume. The first platform that batches and automates this workflow owns the fund admin segment — which is a higher-value, higher-retention customer than individual KVGs.
