Executive Summary

Caelith is a surprisingly deep early-stage RegTech platform. For a pre-revenue startup, the regulatory domain modeling is well above what I'd expect — genuine AIFMD expertise baked into code, not bolted on. The Annex IV XML serializer with XSD schemas, the multi-NCA portal database, LEI verification via GLEIF, investor classification engine with KAGB/ELTIF/EuVECA logic, integrity hash chains on decision records, and an AI copilot with PII stripping are real differentiators.

The hard truth: The platform has strong regulatory depth but lacks production hardening. No CI test runs, no monitoring/observability, no automated deployment pipeline, no real XSD validation (native module optional), mock screening as default, and zero frontend tests. The codebase is a credible MVP — but it's 30% of what a paying KVG would demand.

Biggest risk: The gap between "impressive demo" and "auditable production system" is 6-9 months of focused engineering. Investors will see the depth; regulators will see the gaps.

🚨 Critical Gaps — Top 5

1. No Tests in CI BLOCKER

CI only runs typecheck. 30+ test files exist but are never executed in the pipeline. Any regression, broken migration, or API contract violation ships to production undetected. For a RegTech product where bugs = compliance failures = regulatory fines, this is existential.

Fix: Add npm test and npm run test:e2e to CI. Target: 1 day. Block all merges on green tests.

2. No Monitoring or Observability BLOCKER

Zero monitoring in production. No error tracking confirmed active, no uptime checks, no performance metrics. If the Railway instance goes down at 2 AM before a filing deadline, nobody knows until a KVG calls.

Fix: Sentry for errors (config exists, verify active). UptimeRobot for availability. Structured JSON logs to Railway's log drain. Target: 2 days.

3. Mock Screening as Default HIGH

Without an OpenSanctions API key OR local sanctions data, the system returns mock results. A compliance officer relying on "clear" results from mock mode is a regulatory disaster. There's no UI indicator distinguishing mock from real screening.

Fix: Add prominent "DEMO DATA" badge on mock results. Block production deployment without API key or local data. Target: 1 day.

4. No Actual NCA Filing Integration HIGH

The system generates Annex IV XML but has zero integration with any NCA portal. The value proposition stops at "generate and download." For a product claiming to automate regulatory reporting, this is the biggest functional gap.

Fix: BaFin MVP portal API integration first (largest German market). Then CSSF eReg. Target: 3-6 months for first NCA.

5. No XSD Validation in Default Install HIGH

Real XSD validation requires libxmljs2 (native C++ addon) which isn't a default dependency. The fallback "pattern" validator is incomplete by its own admission. Generated XML may not pass NCA validation.

Fix: Use a WASM-based or pure-JS XML schema validator. Or make libxmljs2 a required dependency with build instructions. Target: 1 week.

🏆 Competitive Position Assessment

Strengths vs Incumbents (Anevis, FundApps, Kneip)

  • Regulatory domain depth at code level — investor classification with KAGB/ELTIF/EuVECA legal citations, ESMA code mappings, NCA-specific grace periods. This isn't just a database with checkboxes; it's compliance logic expressed in code.
  • AI-native architecture — Copilot with tool-use (Claude reads DB schema, writes read-only queries), PII stripping, RAG with regulatory embeddings. Incumbents are bolting AI onto legacy.
  • Open source component — open-annex-iv could become the community standard for Annex IV XML generation. No open source alternative exists today.
  • Modern tech stack — Next.js 15, React 19, TypeScript, PostgreSQL with RLS. Vs incumbents on Java/.NET monoliths.
  • Full-stack bilingual — German/English throughout, including regulatory terminology. Critical for DACH market.

Weaknesses vs Incumbents

  • No production deployments — Zero customers, zero regulatory filings processed. Incumbents have years of NCA relationship and filing history.
  • No NCA portal integration — The "last mile" of regulatory reporting doesn't exist. Generating XML is 40% of the value; submitting it is 60%.
  • No SOC 2 / ISO 27001 — Enterprise KVGs won't sign without compliance certifications. Docs exist (docs/iso27001/) but certification is years away.
  • Solo founder risk — One person building the entire platform. No dedicated security engineer, no compliance domain expert on payroll, no QA.
  • Testing/observability gap — A RegTech product without reliable tests and monitoring is a liability, not an asset.

Verdict

Caelith has better regulatory domain modeling than most Series A RegTech companies I've seen. The investor classification engine alone — with its multi-regime eligibility assessment, legal citation chains, and jurisdiction-aware thresholds — would take a new competitor 6+ months to replicate. The Annex IV serializer with proper ESMA code mappings is production-grade logic wrapped in MVP infrastructure.

The gap is not knowledge or architecture — it's production hardening and the last mile. Fix testing, monitoring, and mock data guardrails (2 weeks). Then focus every resource on BaFin portal integration and getting the first paying pilot. The regulatory depth is the moat; the infrastructure gaps are the walls that need building around it.

Rating: Credible for pilot conversations today. 3-6 months from first production filing. 12-18 months from enterprise-ready.
