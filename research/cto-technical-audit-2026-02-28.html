🔒 Security Posture

72/100
✓ What's Good

Genuinely impressive for 10 weeks. Three-tier rate limiting (API 500/15min, auth 50/15min, export 10/min) with DB-backed store. CSRF origin validation against allowlist. httpOnly/secure/SameSite=lax cookies. bcrypt cost factor 12. JWT 32-char minimum. SSRF guard. Input sanitization with prototype pollution defense. Documented security audit with implemented fixes.

✗ What's Weak

CSP requires unsafe-inline + unsafe-eval (Next.js limitation). IP-based rate limiting = corporate NAT false positives. No WAF/bot detection. No request signing for webhooks.

→ Top Fixes

(1) Nonce-based CSP via Next.js middleware. (2) Per-user authenticated rate limiting.
