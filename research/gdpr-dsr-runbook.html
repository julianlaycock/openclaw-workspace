Art. 15 — Right of Access

Procedure

  1. Verify identity of requester (email confirmation or ID check)
  2. Identify all data linked to the data subject
  3. Generate data export (JSON format)
  4. Include: purposes of processing, categories, recipients, retention periods, rights info
  5. Deliver within 30 days

SQL Queries

-- For a USER (by email):
SELECT id, email, name, role, created_at, updated_at FROM users WHERE email = '[EMAIL]';
SELECT * FROM login_attempts WHERE email = '[EMAIL]';
SELECT * FROM refresh_tokens WHERE user_id = '[USER_ID]';
SELECT * FROM audit_trail WHERE user_id = '[USER_ID]';

-- For an INVESTOR (by name/email):
SELECT * FROM investors WHERE email = '[EMAIL]' OR name ILIKE '%[NAME]%';
SELECT * FROM investor_documents WHERE investor_id = '[INVESTOR_ID]';
SELECT * FROM holdings WHERE investor_id = '[INVESTOR_ID]';
SELECT * FROM onboarding_records WHERE investor_id = '[INVESTOR_ID]';
SELECT * FROM decision_records WHERE subject_id = '[INVESTOR_ID]';
SELECT * FROM transfers WHERE from_investor_id = '[INVESTOR_ID]' OR to_investor_id = '[INVESTOR_ID]';
SELECT * FROM events WHERE entity_type = 'investor' AND entity_id = '[INVESTOR_ID]';

Art. 16 — Right to Rectification

Procedure

  1. Verify identity and receive correction request
  2. Identify incorrect data
  3. Update the record(s)
  4. Log the change in audit_trail
  5. Confirm correction to data subject
-- Update investor details:
UPDATE investors SET name = '[NEW_NAME]', email = '[NEW_EMAIL]', updated_at = NOW()
WHERE id = '[INVESTOR_ID]' AND tenant_id = '[TENANT_ID]';

-- Update user details:
UPDATE users SET name = '[NEW_NAME]', email = '[NEW_EMAIL]', updated_at = NOW()
WHERE id = '[USER_ID]';

-- Always log the change:
INSERT INTO audit_trail (entity_type, entity_id, action, user_id, details, tenant_id, created_at)
VALUES ('investor', '[INVESTOR_ID]', 'gdpr_rectification', '[ADMIN_USER_ID]',
  '{"reason": "GDPR Art 16 request", "fields_changed": ["name", "email"]}',
  '[TENANT_ID]', NOW());

Art. 17 — Right to Erasure

⚠️ Erasure may conflict with legal retention obligations (AML/KYC records, AIFMD regulatory requirements). Document the legal basis if refusing erasure.

Procedure

  1. Verify identity
  2. Check if legal retention obligation applies (AML: 5 years, AIFMD: 5-10 years)
  3. If retention applies → inform data subject of legal basis for refusal
  4. If no retention applies → execute erasure in order (child records first)
  5. Log erasure action in audit_trail
-- Execute in this order to respect foreign keys:
DELETE FROM investor_documents WHERE investor_id = '[INVESTOR_ID]';
DELETE FROM onboarding_records WHERE investor_id = '[INVESTOR_ID]';
DELETE FROM holdings WHERE investor_id = '[INVESTOR_ID]';
UPDATE transfers SET from_investor_id = NULL WHERE from_investor_id = '[INVESTOR_ID]';
UPDATE transfers SET to_investor_id = NULL WHERE to_investor_id = '[INVESTOR_ID]';
DELETE FROM decision_records WHERE subject_id = '[INVESTOR_ID]';
UPDATE events SET metadata = '{"gdpr_erased": true}'
  WHERE entity_type = 'investor' AND entity_id = '[INVESTOR_ID]';
DELETE FROM investors WHERE id = '[INVESTOR_ID]';

-- For users:
DELETE FROM refresh_tokens WHERE user_id = '[USER_ID]';
DELETE FROM password_reset_tokens WHERE user_id = '[USER_ID]';
DELETE FROM login_attempts WHERE email = '[EMAIL]';
-- Anonymize audit trail (retain for compliance, remove PII):
UPDATE audit_trail SET user_id = NULL, details = jsonb_set(details, '{gdpr_erased}', 'true')
  WHERE user_id = '[USER_ID]';
DELETE FROM users WHERE id = '[USER_ID]';

Art. 20 — Right to Data Portability

Procedure

  1. Verify identity
  2. Export all personal data in machine-readable format (JSON)
  3. Include only data provided by the data subject (not derived/computed data)
  4. Deliver within 30 days
-- Generate JSON export:
SELECT json_build_object(
  'personal_data', (SELECT row_to_json(i) FROM investors i WHERE id = '[INVESTOR_ID]'),
  'documents', (SELECT json_agg(row_to_json(d)) FROM investor_documents d WHERE investor_id = '[INVESTOR_ID]'),
  'holdings', (SELECT json_agg(row_to_json(h)) FROM holdings h WHERE investor_id = '[INVESTOR_ID]'),
  'onboarding', (SELECT json_agg(row_to_json(o)) FROM onboarding_records o WHERE investor_id = '[INVESTOR_ID]'),
  'export_date', NOW(),
  'format_version', '1.0'
) AS gdpr_export;

Documentation Requirements

  • Log every DSR in a register: date received, type, data subject identity, completion date, outcome
  • Retain DSR records for 3 years as evidence of compliance
  • If request is refused, document the legal basis
  • All actions must be recorded in the audit_trail table with action type gdpr_*
