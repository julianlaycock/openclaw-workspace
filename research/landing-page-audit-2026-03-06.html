Executive Summary

The Core Problem

⚠️ Caelith's homepage positions it as a compliance dashboard. Sphinx positions itself as a compliance team member. If we're going 100% agents, this is the single biggest thing to fix. Every other recommendation flows from this.

Sphinx says

"Meet Sphinx.
Your last compliance hire."

7 words. Tells you: (1) it's a teammate, not a tool, (2) you'll never hire for this role again, (3) it has a personality.

vs.

Caelith says

"Fund compliance,
automated."

3 words. Tells you: (1) it does compliance, (2) it's automated. That's it. Could be any RegTech company. Zero personality. Zero hook. Zero reason to scroll.

What Sphinx Gets Right

Sphinx's Playbook — Steal These Patterns

1. Agents are people, not features

Every piece of copy frames agents as teammates: "Your last compliance hire," "AI analysts," "Your team of AI compliance analysts." They never say "our software" or "our platform." The agent IS the product. It has a name (Sphinx), a role (compliance analyst), and a job description (clear KYB & AML cases).

Steal this Caelith agents need names or roles. "BaFin Filing Agent" is good. "Your fund's compliance department" is better.

2. Animated demo above the fold

Sphinx shows a live interactive demo right in the hero — screening alerts resolving in real-time with false positives being identified. You SEE the agent working before you read a single paragraph. This is massively effective.

Steal this We have the agent trace animation on /agents. It needs to be on the homepage hero. A compliance officer should see the agent filing to BaFin within 3 seconds of landing.

3. Outcome metrics, not feature lists

Above the fold: 87% fewer false positives, 64% fewer drop-offs, 98% cases resolved same day, 4.2x ops cost reduction. All outcomes. Not "we support 147 Annex IV fields."

Steal this Our hero metrics should be: "25 sec per filing" / "8 funds overnight" / "93% cost reduction" / "0 human reviewers" — outcomes, not capabilities.

4. Before/After kills features pages

"Common Workflow: Compliance still runs on human glue" → visual showing emails, PDFs, manual steps. Then Sphinx workflow → automated, clean, resolved. One image tells the entire story.

Steal this We built this for /agents already. Bring the comparison to the homepage. "8 hours manual vs. 25 seconds with Caelith" is our single strongest selling visual.

5. Real customer stories with real numbers

Named humans (James Simcox, COO), named companies, specific metrics (124 hours/month, 90.8% reduction). This is what VCs and prospects look for. It's proof, not promises.

Can't steal yet We're pre-customer. But we CAN use: grant validation (NGI Zero, Campus Founders), VC thesis validation ($7.1M Sphinx + $12.7M Kobalt), and the Egor Tarakanov quote if we get permission.

6. Single CTA repeated 3 times

"Book a demo" appears in nav, hero, and technology section. Same button. Same action. Same color. No confusion. No "Try Free" vs "Readiness Check" vs "API Docs" fighting for attention.

Critical fix Caelith has 4+ different CTAs competing. Pick ONE primary: "Book a Demo." Everything else is secondary.

7. Trust section is early and visual

SOC 2 Type II badge, GDPR compliance, "100% transparent decisions," "Audit trails your regulator trusts" — all in a dedicated section with cards and iconography. Not tiny pills at the bottom.

Steal this Our trust signals (ESMA XSD, DORA audit trail, EU-hosted, open-source core) are currently tiny pills. They need to be a proper section with visual weight.

Critical Review

What's Wrong With caelith.tech Right Now

🔴 CRITICAL: Hero says nothing about agents

"Fund compliance, automated." — This is a tagline for a dashboard product. We decided to go 100% agents. The homepage has zero mention of agents, AI, or autonomous filing. The /agents page exists but it's completely disconnected from the homepage. A visitor doesn't know agents exist unless they find it in the nav.

Impact: First impression = "another compliance SaaS." Visitors leave in 5 seconds.

🔴 CRITICAL: Terminal/code preview alienates the buyer

The hero shows a code terminal with API responses. Compliance officers don't write code. They're the buyer. The Geschäftsführer at a KVG sees code and thinks "this is for developers, not for me." Sphinx shows a visual screening demo. We show JSON.

Impact: The person who signs the check bounces because the product looks technical, not operational.

🔴 CRITICAL: Too many CTAs = no CTA

Current CTAs on one page: "Start Readiness Check" (hero), "Book a 15-Min Demo" (hero), "See API Docs" (hero), "Try Demo" (multiple), "Readiness Check" (pricing), various nav links. The visitor doesn't know what to click. Choice paralysis = they click nothing.

Fix: ONE primary CTA: "Book a Demo." One secondary: "Check Your Readiness." Everything else goes to nav or footer.

🟡 HIGH: No before/after comparison on homepage

The €180K manual vs €1,890 automated comparison is strong but it's buried in a text section. No visual comparison. No workflow comparison. The /agents page has an excellent before/after grid — it's not on the homepage.

🟡 HIGH: Features section is generic

"Built for compliance. Engineered for speed." — followed by feature descriptions that read like every SaaS product page. "ESMA Annex IV XML in seconds" is good but buried. The features should be framed as agent capabilities, not software features.

🟡 HIGH: Trust signals are weak

Tiny pills that say "ESMA XSD" and "Campus Founders" — most visitors won't even notice them. No dedicated trust/security section. No badges. No visual weight.

🟠 MEDIUM: FAQ is bloated

7 FAQ items on the landing page. Sphinx has zero. FAQ sections signal uncertainty — "we need to explain ourselves." A confident product page answers questions before they're asked, through the copy itself. Keep 3-4 max, move the rest to a /faq page.

🟠 MEDIUM: "Technical guidance" section doesn't belong

"Technical guidance & regulatory insights" is a blog/resources feature, not a landing page section. It dilutes the message. Kill it from the homepage; link to /blog in the nav.

🟢 KEEP: Dashboard screenshot section

"See it in action" with the dashboard preview is good. But reframe it: instead of "look at our dashboard," make it "here's what your agent reports back to you." The dashboard is the agent's reporting interface, not the product itself.

🟢 KEEP: Pricing section

Visible pricing is a differentiator vs. enterprise competitors. The €990/€1,990/€3,500 tiers are clear. Keep but move below the trust section.

Recommended Structure

New Homepage — Section by Section

Section 1: Hero + Agent Animation

Slogan + subtitle + animated agent trace (from /agents page). Show the BaFin filing agent running through its steps live. Metrics bar: 25 sec / 100% XSD / 8 funds / €0 review cost. Single primary CTA: "Book a Demo." Secondary: "Watch the Agent Work →" (scrolls to trace).

Kill Terminal/code preview. Kill "Start Readiness Check" as primary CTA.

Section 2: Trust Bar

Immediately after hero: "ESMA XSD Validated · Campus Founders-Backed · NGI Zero Grantee · EU-Hosted · Open Source Core" — visual badges, not text pills. If we get customer logos later, this becomes the social proof bar.

Section 3: Before / After

The manual process (7 steps, ~16 hours) vs. Caelith Agent (7 steps, 3 min 47 sec). Already built for /agents. This is the most compelling section on that page — bring it to the homepage.

Section 4: Agent Fleet

6 agent cards with Live/Building/Soon badges. Shows breadth: BaFin Filing, CSSF Filing, Investor DD, Sanctions Monitoring, Readiness Assessment, Regulatory Watch. Positions Caelith as a platform, not a single-feature tool.

Section 5: Dashboard Preview

Reframed: "Your agents report back here." Screenshot of the Caelith dashboard. Emphasis on visibility and control — the human stays in charge, agents do the work. CTA: "Try the Demo →" (links to /demo-dashboard).

Section 6: Trust & Security

Dedicated section (like Sphinx's Auditability block). ESMA XSD validation, SHA-256 audit chain, DORA compliance, EU-hosted (Railway Amsterdam), open-source serializer. Visual cards with icons.

Section 7: Pricing

Keep current structure. Add the "€180K manual vs. €1,890" comparison as the section intro. It's the strongest conversion line on the page.

Section 8: Urgency + CTA

Countdown to April 16 AIFMD II deadline + final "Book a Demo" CTA. "41 days until enforcement. Your agent is ready."

Section 9: FAQ (4 questions max)

Only the 4 questions compliance officers actually ask: (1) "How does the agent file to BaFin?" (2) "Is the XML ESMA-validated?" (3) "Where is my data stored?" (4) "Can I try it before buying?"
