## 4. Technical SEO Checklist

### 4A. Immediate (Do TODAY)

#### robots.txt
Create `public/robots.txt`:
```
User-agent: *
Allow: /

Sitemap: https://www.caelith.tech/sitemap.xml
```

#### sitemap.xml
Use `next-sitemap` package:

```bash
npm install next-sitemap
```

Create `next-sitemap.config.js`:
```js
/** @type {import('next-sitemap').IConfig} */
module.exports = {
  siteUrl: 'https://www.caelith.tech',
  generateRobotsTxt: true,
  changefreq: 'weekly',
  priority: 0.7,
  sitemapSize: 5000,
  alternateRefs: [
    {
      href: 'https://www.caelith.tech/de',
      hreflang: 'de',
    },
  ],
  transform: async (config, path) => {
    // Higher priority for key pages
    const highPriority = ['/', '/aifmd-ii', '/readiness-check', '/blog'];
    return {
      loc: path,
      changefreq: path.startsWith('/blog/') ? 'monthly' : 'weekly',
      priority: highPriority.includes(path) ? 1.0 : 0.7,
      lastmod: new Date().toISOString(),
    };
  },
};
```

Add to `package.json`:
```json
"scripts": {
  "postbuild": "next-sitemap"
}
```

#### Canonical URLs
In `_app.tsx` or layout, add to `<Head>`:
```jsx
<link rel="canonical" href={`https://www.caelith.tech${router.asPath.split('?')[0]}`} />
```

#### Hreflang Tags
Add to every page `<Head>`:
```jsx
<link rel="alternate" hreflang="en" href={`https://www.caelith.tech${path}`} />
<link rel="alternate" hreflang="de" href={`https://www.caelith.tech/de${path}`} />
<link rel="alternate" hreflang="x-default" href={`https://www.caelith.tech${path}`} />
```

#### Open Graph / Twitter Cards
Add to every page:
```jsx
<meta property="og:title" content={title} />
<meta property="og:description" content={description} />
<meta property="og:image" content="https://www.caelith.tech/og-image.png" />
<meta property="og:url" content={canonicalUrl} />
<meta property="og:type" content="website" />
<meta property="og:site_name" content="Caelith" />
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content={title} />
<meta name="twitter:description" content={description} />
<meta name="twitter:image" content="https://www.caelith.tech/og-image.png" />
```

```tsx
// app/nca/[code]/page.tsx
import { ncaData } from '@/data/ncas';

export function generateStaticParams() {
  return Object.keys(ncaData).map(code => ({ code }));
}

export function generateMetadata({ params }) {
  const nca = ncaData[params.code];
  return {
    title: `AIFMD II Reporting to ${nca.name}: ${nca.country} Fund Manager Guide | Caelith`,
    description: `Complete guide to filing AIFMD II reports with ${nca.name}. Requirements, deadlines, XML format, and filing portal for ${nca.country} fund managers.`,
    alternates: {
      canonical: `https://www.caelith.tech/nca/${params.code}`,
    },
  };
}
```

### Homepage FAQ (Add These to Homepage with FAQPage Schema)

**Q: What is AIFMD II?**
A: AIFMD II (Directive 2024/927/EU) is the revision of the EU Alternative Investment Fund Managers Directive. It introduces new requirements for delegation, liquidity management, loan origination, and regulatory reporting. Enforcement begins April 16, 2026.

**Q: When is the AIFMD II deadline?**
A: The AIFMD II enforcement deadline is April 16, 2026. All EU AIFMs must comply with the new requirements by this date.

**Q: What is Annex IV reporting?**
A: Annex IV is the standardized AIFMD reporting template. Fund managers must file XML reports with their national regulator (BaFin, CSSF, AMF, etc.) containing fund data validated against ESMA's XSD schema.

**Q: Is Caelith free?**
A: Caelith's core is open source (Apache 2.0). The readiness check and Annex IV XML validation are free. Advanced features like automated sanctions screening and audit trails are available in paid plans.

**Q: Which regulators does Caelith support?**
A: Caelith supports filing with BaFin (Germany), CSSF (Luxembourg), FMA (Austria), AMF (France), CBI (Ireland), and other EU national competent authorities. All Annex IV output is ESMA XSD-validated.
