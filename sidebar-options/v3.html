Sidebar v3 — 4 Design Directions

Same fundamental structure (Dashboard → Portfolio → Compliance → Monitor). Four different design philosophies. Each one solves a specific problem with the current sidebar.

A — Weighted Clarity

Search-first. Signal with restraint.
Recommended

Search moves to the top — it's the #1 action in a compliance tool. Removed section counts. Added subtle status dots on items that need attention. Active bar indicator. Maximum clarity, minimum noise.

What changed:
  • Search → top — #1 user action gets #1 position
  • No quick action buttons — they competed for attention and ⌘K does the same job better
  • No section counts — removed noise
  • Status dots — amber on Investors (KYC expiring), Decisions (pending), Calendar (upcoming deadlines). Green on Readiness (all clear). Sidebar now communicates
  • Active bar — 3px left accent, subtle but unmistakable
Best for: Teams who live in this tool daily. Clean, informative, no gimmicks.

B — Signal-Rich

The sidebar tells you what's wrong.
Compliance-native

Alert banner at top when things need attention. Inline count badges on items. The sidebar is not just navigation — it's your first line of defense. You see problems before you click.

What changed:
  • Alert banner — "3 items need attention" pulls from real data (KYC expiring, pending decisions, upcoming deadlines)
  • Inline count badges — Investors (1 KYC expiring), Decisions (2 pending), Calendar (2 deadlines). Numbers on the items that matter
  • No quick actions, no search bar — search icon in footer, ⌘K for power users
  • Tighter spacing — more items visible without scrolling
Best for: Compliance-heavy orgs where the sidebar is a monitoring surface, not just navigation.

C — Breathe

Less is more. Progressive disclosure.
Minimal

Maximum whitespace. Sub-items appear on hover. Command palette (⌘K) replaces quick actions. Section labels use sentence case, not all-caps mono — warmer, more approachable.

What changed:
  • "Search or jump to…" — combined search + command palette. One box to rule them all
  • Sub-items on hover — Funds children appear when you hover. Reduces visual noise by ~40%. Active section stays expanded
  • Sentence-case labels — "Portfolio" not "PORTFOLIO". Less aggressive, more approachable
  • Generous padding — items breathe. Easier to click, easier to scan
  • No badges, no dots — the dashboard handles alerting. Sidebar stays calm
Best for: Demos, first impressions, smaller teams. Looks premium but hides depth.

D — Power Dashboard

Information density for pros.
Dense

Mini status bar shows portfolio health at a glance. Section-level attention badges. Tighter spacing, smaller type. Built for the compliance officer who has 30 tabs open and needs to see everything.
